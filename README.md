# 互联网实习生模拟器

一个互动式的文字模拟小游戏，从 **求职投简历 → 笔试面试 → 入职实习 → 转正博弈**，体验互联网实习生的一整条路径。

> 每一个选择都会改变：生命值、精力、声誉、压力、技能、钱包，以及你的剧情走向。

---

## 功能亮点

- 🎭 **多种人设可选**
  - 985 研究生（技术岗）
  - 985 本科（技术岗）
  - AI 方向研究生（技术岗）
  - 双非市场营销（`role: marketing`）
  - 海归运营（`role: operations`）
  - 产品岗应届生（`role: product`）

- 🧩 **岗位专属剧情**
  在不改变底层框架的前提下，为不同 `role` 设计了独立风格的题目与文案：
  - 市场营销：品牌推广、拉新活动、种草文案等娱乐化情境。
  - 运营：社群活跃、活动复盘、留存/转化数据分析。
  - 产品：用户痛点、功能设计、需求文档/原型/评审会。
  - 技术：继续走原本「通用」线路，偏技术/泛互联网场景。

  代码层面仍然使用同一个 `HUNT_EVTS` / `INTERN_EVTS` 事件数组，只是在 `curEvts()` 中根据 `G.char.role` 选择是否应用 `EVT_OVERRIDES` 做文案和选项覆盖，所以整体状态机逻辑保持不变。

- 📊 **六维数值系统**
  - ❤️ 生命（状态）
  - ⚡ 精力
  - ⭐ 声誉
  - 🌀 压力
  - 📚 技能
  - 💰 钱包

  每个选项都会带来不同的数值变化，部分事件还会根据你的岗位人设给出**额外加成或惩罚**。

- 🧠 **人设动态加成**
  通过 `applyCharBonus` 在不改动核心 `applyD` 的前提下，对不同角色做细节调整：
  - 985 / AI 角色：在技术/专业场景中额外获得技能或声誉。
  - 市场人设：在营销相关事件中，声誉成长更快。
  - 产品人设：在用户洞察、需求评审类事件中，额外加声誉。
  - 双非人设：在「突破」「逆袭」场景选到高风险高收益选项时会有额外正向加成。
